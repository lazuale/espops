# 00. Глоссарий

[К содержанию](../README.md) · [Далее: установка](01-install.md)

Короткий словарь терминов, которые встречаются в документации. Этого достаточно, чтобы понимать инструкции, даже если раньше вы не работали с Docker.

## Docker и контейнеры

| Термин | Простыми словами |
|---|---|
| Docker | программа, которая запускает приложения в изолированных «коробках» — контейнерах. Избавляет от ручной установки зависимостей на сервер. |
| Образ (image) | готовый шаблон приложения определённой версии, например `espocrm/espocrm:9.3.7-apache-trixie`. Из образа создаётся контейнер. |
| Контейнер (container) | запущенный экземпляр образа. В этом проекте их четыре: веб-контейнер, база, фоновый обработчик и WebSocket. |
| Том (volume) | отдельное хранилище Docker для данных. Данные EspoCRM лежат в томах, а не в каталоге проекта, поэтому переустановка контейнеров их не удаляет. |
| Docker Compose | инструмент, который запускает сразу несколько связанных контейнеров по описанию из файла `docker-compose.*.yml`. |
| compose-файл | файл с описанием контейнеров стенда: `docker-compose.prod.yml` (рабочий) и `docker-compose.dev.yml` (тестовый). |
| healthcheck | встроенная проверка готовности контейнера. Пока контейнер не «здоров» (`healthy`), зависящие от него контейнеры ждут. |

## Файлы и настройки проекта

| Термин | Простыми словами |
|---|---|
| `.env.prod` / `.env.dev` | файлы с настройками и паролями рабочего и тестового стендов. В Git не попадают. |
| `.env.prod.example` | шаблон настроек из репозитория. Реальный `.env.prod` создаётся из него на сервере. |
| `data/config.php` | основной конфиг EspoCRM внутри тома: адрес стенда и другие настройки. |
| `data/config-internal.php` | внутренний конфиг EspoCRM: параметры подключения к базе (имя, пользователь, пароль). |
| репозиторий (Git) | хранилище файлов проекта на GitHub, откуда сервер забирает обновления командой `git pull`. |

## Стенды

| Термин | Простыми словами |
|---|---|
| рабочий стенд (prod) | основной EspoCRM, которым пользуются люди. Открывается по `http://10.0.2.7`. |
| тестовый стенд (dev) | отдельная копия для экспериментов. Открывается по другому порту и не влияет на рабочий стенд. |
| перенос prod → dev | заливка свежей копии рабочего стенда в тестовый, чтобы проверить обновления и изменения без риска. |

## Резервное копирование

| Термин | Простыми словами |
|---|---|
| резервная копия (backup) | архив с базой и файлами EspoCRM на определённый момент времени. Нужен для отката и восстановления. |
| восстановление (restore) | замена текущих данных стенда данными из резервной копии. |
| архив `.tar.gz` | сжатый файл. Резервная копия — это один такой архив с именем вида `2026-01-31_010000.tar.gz`. |

## Команды и доступ

| Термин | Простыми словами |
|---|---|
| SSH | способ удалённо подключиться к серверу и выполнять команды в терминале. |
| `sudo` | запуск команды с правами администратора сервера. |
| порт | «дверь» сети, по которой доступна служба. Рабочий стенд использует порт `80`, тестовый — `8080`. |
