# 06. Чек-листы

[Назад: обновление](05-update.md) · [К содержанию](../README.md) · [Далее: типичные проблемы](07-troubleshooting.md)

Чек-листы нужны для контроля перед важными действиями. Они не заменяют инструкции, а помогают не пропустить обязательные проверки.

## 1. Первый запуск рабочего стенда

Перед запуском:

| Проверка | Готово |
|---|---|
| Docker, Docker Compose и Git установлены, `docker ps` работает без `sudo` | [ ] |
| Репозиторий склонирован в `/opt/espocrm` | [ ] |
| Создан `.env.prod` | [ ] |
| Пароли заменены и записаны в одинарных кавычках | [ ] |
| Права на `.env.prod`: `600` | [ ] |
| Порты `80` и `8081` свободны | [ ] |
| `docker compose config` проходит без ошибок | [ ] |

После запуска:

| Проверка | Готово |
|---|---|
| Контейнеры `espocrm-db`, `espocrm`, `espocrm-daemon`, `espocrm-websocket` запущены | [ ] |
| `espocrm` и `espocrm-db` дошли до статуса `healthy` | [ ] |
| Сайт открывается по `http://10.0.2.7` | [ ] |
| Порт `8081` доступен | [ ] |
| Вход администратора работает | [ ] |
| Сделана первая резервная копия | [ ] |
| `.env.prod` сохранён отдельно в защищённом месте | [ ] |

## 2. Резервная копия

| Проверка | Готово |
|---|---|
| Команда выполняется из `/opt/espocrm` | [ ] |
| Каталог `backups/` существует | [ ] |
| Скрипт `scripts/backup-docker-container.sh` на месте | [ ] |
| Образ `alpine` доступен или может быть скачан | [ ] |
| Архив `YYYY-MM-DD_HHMMSS.tar.gz` создан | [ ] |
| Архив читается через `tar tzf` | [ ] |
| Внутри есть `db.tar.gz` и `files.tar.gz` | [ ] |
| Важная копия вынесена за пределы сервера | [ ] |
